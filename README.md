# Аналитика продаж OZON: кабели и зарядные устройства (2024–2026)

Этот проект посвящён анализу продаж кабелей и зарядных устройств на маркетплейсе OZON за период 2024–2026 годов. 
Цель — выявить динамику спроса, ценовые тенденции, сезонность, ключевые категории товаров и факторы, влияющие на продажи.

## Структура проекта

Проект организован по принципам аналитики данных и включает следующие разделы:

- `sql/` — SQL-скрипты для создания таблиц, загрузки данных и аналитических запросов.
- `python/` — скрипты для ETL-процессов (загрузка и очистка данных), а также файлы с EDA (исследовательским анализом данных).
- `powerbi/` — итоговый интерактивный дашборд Power BI, визуализирующий ключевые метрики и результаты анализа.
- `docs/` — документация проекта: словарь данных, архитектура, описание витрин, гипотезы и план анализа.
- `data/` — ссылки на исходные датасеты, размещённые в Google Drive (файлы данных не хранятся в репозитории).

## Данные

Исходные данные представляют собой выгрузки с маркетплейса OZON за период 2022–2024 годов. 
Они включают информацию о продажах, остатках, ценах, скидках и характеристиках товаров.

В проекте используются следующие наборы данных:

- **sales.csv** — данные о продажах (дата, товар, количество, цена, скидка, выручка).
- **stock.csv** — данные об остатках товаров на складах.
- **prices.csv** — история цен и промо-акций.
- **products.csv** — справочник товаров (категория, бренд, характеристики).

Файлы данных не хранятся в репозитории из-за их объёма. 
Все исходные датасеты размещены в Google Drive, ссылки находятся в папке `data/`.

## Архитектура проекта

Проект построен по принципу аналитического конвейера (data pipeline), где каждый этап отвечает за свою часть обработки данных:

1. **CSV → Python (ETL)**  
   - загрузка исходных файлов;
   - очистка данных (дубликаты, пропуски, типы данных);
   - нормализация и подготовка к загрузке в БД.

2. **Python → PostgreSQL**  
   - создание таблиц;
   - загрузка очищенных данных в базу;
   - формирование витрин для анализа.

3. **PostgreSQL → Python (EDA)**  
   - исследовательский анализ данных (EDA);
   - визуализации (seaborn, matplotlib);
   - статистические тесты (scipy);
   - расчёт метрик и выявление закономерностей.

4. **PostgreSQL → Power BI**  
   - подключение к витринам;
   - построение интерактивного дашборда;
   - визуализация ключевых метрик и выводов.

## Используемые технологии

В проекте применяются инструменты и библиотеки, характерные для работы аналитика данных:

**Язык программирования:**
- Python 3.10+

**Библиотеки Python:**
- pandas — обработка и анализ данных;
- numpy — численные вычисления;
- seaborn, matplotlib — визуализация;
- scipy — статистические тесты;
- SQLAlchemy / psycopg2 — работа с PostgreSQL.

**База данных:**
- PostgreSQL 14+ — хранение данных, витрины, аналитические запросы.

**BI-инструмент:**
- Power BI — интерактивный дашборд и визуализация ключевых метрик.

**Система контроля версий:**
- Git + GitHub — управление проектом и документацией.
