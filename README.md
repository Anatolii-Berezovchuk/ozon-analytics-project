# Аналитика будущего рынка труда в эпоху ИИ (2020–2026)

Этот проект посвящён анализу глобальных изменений на рынке труда под влиянием искусственного интеллекта.  
Используется датасет **Future of Work in the Age of AI (2020–2026)** с Kaggle, содержащий более 15 000 строк данных о профессиях, отраслях, странах, уровне автоматизации, рисках замещения, зарплатах и динамике изменений.

Цель проекта — выявить тенденции трансформации рынка труда, определить наиболее уязвимые профессии, оценить влияние ИИ на отрасли и страны, а также выделить ключевые навыки будущего.

---

## Структура проекта

Проект организован по принципам аналитики данных и включает следующие разделы:

- `sql/` — SQL-скрипты для создания таблиц, загрузки данных и аналитических запросов.
- `python/` — скрипты для ETL-процессов (загрузка и очистка данных), а также файлы с EDA (исследовательским анализом данных).
- `powerbi/` — итоговый интерактивный дашборд Power BI, визуализирующий ключевые метрики и результаты анализа.
- `docs/` — документация проекта: словарь данных, архитектура, описание витрин, гипотезы и план анализа.
- `data/` — ссылки на исходный датасет Kaggle (файл не хранится в репозитории из-за ограничений лицензии).

---

## Данные

В проекте используется датасет **Future of Work in the Age of AI (2020–2026)** с Kaggle.  
Он содержит информацию о:

- профессиях и отраслях;
- странах и регионах;
- уровне автоматизации;
- риске замещения ИИ;
- средней зарплате;
- требуемых навыках;
- динамике изменений с 2020 по 2026 год.

Примерные поля датасета:

- `occupation` — профессия  
- `industry` — отрасль  
- `country` — страна  
- `automation_risk` — риск автоматизации  
- `ai_impact_score` — влияние ИИ  
- `salary` — средняя зарплата  
- `skills` — ключевые навыки  
- `year` — год наблюдения  

Файл данных размещён в Kaggle, ссылка находится в папке `data/`.

## Архитектура проекта

Проект построен по принципу аналитического конвейера (data pipeline), где каждый этап отвечает за свою часть обработки данных:

### 1. **CSV → Python (ETL)**  
- загрузка исходного файла;
- очистка данных (дубликаты, пропуски, типы данных);
- нормализация и подготовка к загрузке в БД.

### 2. **Python → PostgreSQL**  
- создание таблиц;
- загрузка очищенных данных в базу;
- формирование витрин для анализа.

### 3. **PostgreSQL → Python (EDA)**  
- исследовательский анализ данных (EDA);
- визуализации (seaborn, matplotlib);
- статистические тесты (scipy);
- выявление закономерностей:
  - профессии с высоким риском автоматизации  
  - отрасли, наиболее подверженные влиянию ИИ  
  - динамика зарплат  
  - анализ навыков будущего  

### 4. **PostgreSQL → Power BI**  
- подключение к витринам;
- построение интерактивного дашборда;
- визуализация ключевых метрик:
  - риск автоматизации  
  - влияние ИИ  
  - зарплаты  
  - отрасли  
  - страны  
  - навыки

## Используемые технологии

В проекте применяются инструменты и библиотеки, характерные для работы аналитика данных:

### **Язык программирования:**
- Python 3.10+

- PostgreSQL 14+ — хранение данных, витрины, аналитические запросы

### **BI-инструмент:**
- Power BI — интерактивный дашборд и визуализация ключевых метрик

### **Система контроля версий:**
- Git + GitHub — управление проектом и документацией
